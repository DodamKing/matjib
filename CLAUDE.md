# matjib — AI 결정장애 치료기 (식당 편)

> 선택지를 늘리지 않고 **극단적으로 줄여서** 결정을 대신 내려주는 모바일 웹 서비스.
> "더 이상 고르지 마세요. 걸어서 5분 거리 맛집, AI가 딱 3개만 처방합니다."

상세 기획: `docs/Decision_Disorder_Cure_Plan.pdf`

문서 지도:
- `docs/DECISIONS.md` — 왜(결정·근거)
- `docs/ROADMAP.md` — 무엇을 어떤 순서로 (빌드 계획)
- `docs/ARCHITECTURE.md` — 구조·데이터 흐름·타입
- `docs/MVP_SCOPE.md` — 만들 것 / 안 만들 것
- `docs/WORKLOG.md` — 실제 진행 상황 (**새 세션은 이걸 먼저 읽을 것**)

## 기술 스택 (확정)

- **Next.js (App Router) + TypeScript** — 프론트 + API Route 한 프로젝트
- **Tailwind CSS** — 모바일 우선, 미니멀 카드 UI
- **데이터**: 소상공인시장진흥공단 상가(상권)정보 API (공공데이터포털)
- **AI 매칭**: Anthropic Claude API — 자연어 상황 → 업종 분류 "번역" (맛 판단 아님)
- **지도/길찾기 보강**: 카카오 로컬 API (선택적, 상호명+좌표 매칭)
- **배포 대상**: Vercel

## 핵심 제품 원칙 (= 하지 말 것 목록)

코드를 짤 때 아래를 어기면 제품 정체성이 깨집니다.

- **Zero-Scroll**: 리스트/무한스크롤 금지. 항상 카드 **3개**만.
- **원클릭 진입**: 회원가입·로그인·앱설치 없음. 위치 동의 한 번이면 끝.
- **품질 랭킹 안 함**: 평점/리뷰로 "맛집 순위"를 매기지 않는다. 정렬은
  *거리 + 업종 적합도 + 공정 셔플(랜덤)*. (데이터에 품질 신호가 없음 — `DECISIONS.md` D3)
- **AI는 맛 평가자가 아니라 번역기**: 자연어 → 업종/키워드 변환까지만.
- **광고/협찬 없음**: 상위노출·스폰서 카드 개념 자체를 두지 않는다.

## 보안 규칙 (필수)

- 모든 외부 API 키(공공데이터, Anthropic, 카카오)는 **서버(API Route)에서만** 사용.
  클라이언트 번들에 키를 절대 노출하지 말 것. `NEXT_PUBLIC_` 접두사 금지(키에 한해).
- 키는 `.env.local`에 두고 `.gitignore`로 제외. `.env.example`만 커밋.

## 작업 규칙

- **dev 서버는 사용자가 직접 띄운다.** Claude는 `npm run dev` 등 서버 실행을 하지 않는다.
- **git push는 사용자가 직접 한다.** Claude는 push 하지 않는다.
- **커밋은 요청할 때만, 묶어서 한다.** 변경마다 자잘하게 커밋하지 말고, 사용자가 요청하는 시점에
  그동안의 관련 변경을 의미 단위 한 커밋으로 묶는다 (코드 + 문서 함께).
- **모든 기록은 `docs/`에 문서화한다.** 공유를 위해 Claude 개인 메모리 시스템은 쓰지 않는다.
- 커밋 메시지에 `Co-Authored-By: Claude` 등 AI 태그 **추가 금지**. 사용자 본인 명의로만.
- 새 기능을 만들기 전에 `docs/MVP_SCOPE.md`의 "안 만드는 것" 목록을 먼저 확인.

### 문서 동기화 (필수)
**모든 작업을 마치면, 그 작업으로 낡아진 문서를 전부 현재 상태와 일치시킨 뒤 마무리한다.**
"작업 끝 = 코드 + 문서 둘 다 현재와 일치" 가 완료 기준이다. 아래를 매번 점검:

| 문서 | 갱신 트리거 |
|---|---|
| `docs/WORKLOG.md` | **항상** — 맨 위에 (한 일 / 현재 상태 / 다음 할 것) |
| `docs/STRUCTURE.md` | 파일 추가·삭제·이동, 책임/주요 export·구현상태(✅🟡⬜) 변경 시 |
| `docs/DECISIONS.md` | 설계가 바뀌는 결정을 내렸을 때 한 줄 추가 |
| `docs/ROADMAP.md` | Phase 항목 완료/추가 시 체크박스·내용 갱신 |
| `docs/MVP_SCOPE.md` | 만들 것/안 만들 것 경계가 바뀌었을 때 |
| `README.md` | 공개용 — 사용자에게 보이는 진행 상황/기능이 바뀌는 페이즈 경계에서 갱신 |
| `docs/ARCHITECTURE.md` | 폴더 구조·데이터 흐름·타입이 실제와 달라졌을 때 |

토큰 효율의 핵심 — 다음 세션이 코드를 뒤지지 않고 문서만으로 이어가게 한다.
